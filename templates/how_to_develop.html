Development Guide

Website Filesystem

Each new website has a specific folder structure. Placing your files correctly is essential for the site to work.

  • index.html: The main HTML file for your site's content.
  • /static/css/style.css: The primary stylesheet for all your CSS rules.
  • /static/js/main.js: The main JavaScript file for your site's logic.
  • /static/images/: Upload your images to this folder using the File Manager.

Linking Images

To display an image, you must use the correct path from within your index.html file.

For an image named logo.png in the images folder, use:

<img src="/static/images/logo.png">

Backup & Restore Feature

The Live Editor automatically creates backups as you save. This feature has some important limitations:

What IS backed up?

The system only creates automatic backups for the three core files used in the Live Editor:

  • index.html
  • style.css
  • main.js

What is NOT backed up?

Automatic backups do not cover any other files, including:

  • Images or other assets.
  • Any other .html, .css, or .js files you create manually.
  • Folders and other miscellaneous files.

Pro Tip:

For a complete backup of your entire site, including images and all other files, use the "Download Backup (ZIP)" button from the dashboard.
